## 摘要

针对普通摄像头场景下车辆危险状态识别成本较高、部署复杂的问题，本文设计并实现了一套简易视频监控下的车辆危险状态识别与实时预警工具。系统以摄像头或本地视频文件作为输入，采用 HLS 颜色空间阈值分割、Sobel 边缘检测、透视变换、直方图滑窗搜索和二阶多项式拟合完成车道线检测；采用 Shi-Tomasi 角点检测与 Lucas-Kanade 稀疏光流法估算车辆运动速度；根据车道线顶部中点相对图像中心的偏移估计车辆转向趋势；当速度或角度超过设定阈值时触发页面预警，并将记录写入 SQLite 数据库。系统后端基于 Python、Flask 和 OpenCV 实现，前端采用 HTML、CSS 和 JavaScript 构建，能够完成实时视频显示、状态查看、阈值设置、视频源切换和历史记录查询等功能。

测试结果表明，在 1280×720 分辨率视频条件下，系统离线处理帧率超过 56 fps，达到任务书提出的不低于 30 fps 的实时处理要求。系统能够实现车道线检测、速度估算、危险状态提示和 Web 端展示等基本功能，但在光照变化较大、车道线磨损严重或参数标定不足时，检测效果会出现下降。总体来看，本文完成的系统实现了从算法处理到浏览器端展示的完整流程，可作为低成本车辆危险状态识别与预警的原型验证工具。

关键词：车道线检测；单目视觉；光流测速；实时预警；Flask

## Abstract

This paper designs and implements a low-cost real-time warning system for vehicle dangerous-state recognition based on monocular video. The system takes a camera stream or local video file as input. Lane detection is achieved through HLS thresholding, Sobel edge extraction, perspective transform, histogram sliding-window search, and second-order polynomial fitting. Vehicle speed is estimated by Shi-Tomasi feature detection and the Lucas-Kanade sparse optical flow method. The turning tendency of the vehicle is inferred from the offset between the lane center and the image center. Warnings are triggered when the estimated speed or angle exceeds preset thresholds, and warning records are stored in a SQLite database. The backend is developed with Python, Flask, and OpenCV, while the frontend is implemented using HTML, CSS, and JavaScript.

Experimental results show that the offline processing speed exceeds 56 fps for 1280×720 videos, which satisfies the real-time requirement of the project. The system can complete basic functions including lane detection, speed estimation, and warning display. However, its performance still degrades in scenes with large illumination changes, worn lane markings, or insufficient calibration. Overall, the proposed system completes a full workflow from algorithm processing to Web-based visualization, and can serve as a prototype for low-cost vehicle dangerous-state warning applications.

Key Words: lane detection; monocular vision; optical flow speed estimation; real-time warning; Flask

## 第1章 绪论

### 1.1 研究背景与意义

道路交通场景中，超速行驶、急转向以及驾驶员反应不及时等情况均可能引发安全风险。如果系统能够在车辆运行过程中对异常状态进行及时识别并发出预警，将有助于降低部分交通事故发生的概率。现有驾驶辅助系统虽然已经较为成熟，但商用方案通常依赖毫米波雷达、激光雷达、高精度定位设备以及惯性传感器，整体成本较高，系统结构也较为复杂，不利于在校园通勤车辆、普通私家车或教学实验环境中快速复现。

相比之下，基于单目摄像头的视频分析方案具有硬件门槛低、部署方式灵活和实现流程直观等特点。传统图像处理方法虽然在复杂环境中的鲁棒性不及深度学习方法，其算法链路清晰，运行效率较高，能够满足快速车道检测与预警功能的应用需求。将车道线检测、视觉测速、阈值预警与 Web 可视化展示整合到同一系统中，有助于形成从算法实现到系统部署的完整研究过程。

因此，本文面向低成本场景，设计并实现一套简易视频监控下的车辆危险状态识别与实时预警工具。本文工作的意义主要体现在两个方面：一方面，验证传统视觉方法在简化道路场景中的可行性，并考察其在普通计算平台上达到 30 fps 实时处理目标的能力；另一方面，构建一套具备实时显示、参数配置和历史记录查询功能的 Web 端原型系统，为后续优化研究提供基础。

### 1.2 国内外研究现状

#### 1.2.1 车道线检测研究现状

车道线检测是驾驶辅助系统的基础任务。早期方法以灰度化、边缘检测、霍夫变换和几何拟合为主，实现简单，运算速度快，在普通硬件上也能运行。

后来，透视变换、颜色阈值分割和滑窗搜索逐渐被引入检测流程，可以在鸟瞰图视角下做更稳定的拟合，弯道场景下比直线检测更实用。

深度学习方法在精度上有明显提升。Qin 等提出的 Ultra Fast Structure-Aware Deep Lane Detection 在公开数据集上表现较好[1]；Zheng 等的 CLRNet 通过跨层特征融合进一步提升了检测效果[2]；国内也有研究针对多场景泛化问题提出了结合 GLNet 的改进方案[3]。但这类方法依赖大量训练数据和较高算力，普通硬件上部署成本较高。

本文没有采用深度学习路线，而是用传统图像处理方法完成一个可运行的原型，虽然精度不如深度学习方法，但是成本更低，性能已经能够满足本文的使用需求。

#### 1.2.2 视觉测速研究现状

车辆测速常见方法有雷达、GPS、车载传感器和视觉测速。前几种精度高，但需要额外硬件。视觉测速只需摄像头，部署门槛低，适合低成本场景。

基本思路是用连续帧之间的像素位移，结合帧率和尺度参数换算实际速度。常见方法包括目标跟踪、光流法、特征点匹配和地面参考物换算。Lian 等人在单目测速研究中分析了像素距离与实际速度的映射关系及误差来源[4]，说明在有基本标定条件的前提下，单目视觉可以给出可用的速度估计。

视觉测速容易受安装位置、路面纹理、摄像机抖动和标定误差影响，更适合做辅助判断，不能替代专业测速设备，但在本文研究中已能够满足使用需求。本文引入光流测速模块，主要用于判断车辆状态变化，与转向趋势角一起构成预警依据。

#### 1.2.3 驾驶预警系统研究现状

驾驶预警系统研究通常围绕疲劳驾驶、车道偏离、前碰撞、超速和异常转向展开。商用系统往往包含多传感器融合、环境建模和复杂决策逻辑。学术研究中也有不少只针对单一问题的方案，例如单一车道偏离检测或驾驶员状态识别。

#### 1.2.4 现有研究评述

综合已有研究可以看出，车道线检测、视觉测速和驾驶预警均已形成较为成熟的研究基础。深度学习方法在复杂场景下具有较高精度，但训练和部署成本较高；传统图像处理方法虽然在鲁棒性方面存在一定不足，但实现简单、运行速度快，更适合低成本原型系统开发。基于此，本文选择传统视觉方法作为主要技术路线，重点研究如何在单目视频条件下完成车道线检测、速度估算、危险状态判断与 Web 端展示，并通过系统实现验证该方案的可行性。结合开题报告与任务书要求，本文的实现重点放在"实时检测、阈值预警、浏览器端使用"三个方面。

### 1.3 本文研究内容

结合开题报告、任务书要求与系统实现内容，本文主要完成以下工作：

1. 设计基于 HLS 阈值、Sobel 边缘、透视变换和滑窗搜索的车道线检测流程。
2. 利用特征点和光流法建立基于像素位移的速度估算方法。
3. 根据左右车道线顶部位置的偏移估算转向趋势角，并与速度阈值共同组成危险状态判断逻辑。
4. 使用 Flask 构建简易后端，提供实时视频流、状态查询、阈值设置、视频源切换和历史记录查询接口。
5. 使用 HTML、CSS 和 JavaScript 实现浏览器端页面，展示实时检测结果和历史预警记录。

前期方案中给出了基于 OpenCV 的传统图像处理、危险状态自动预警以及 Web 端工具构建等明确目标。在论文实际实现过程中，总体目标不变的前提下，我对部分具体技术路线进行了调整。初始方案中曾考虑采用 Canny 边缘检测、霍夫变换、FastAPI 与 WebSocket 等实现方式，而在后续实验中，结合当前视频场景与代码调试结果，最终选用了 HLS 阈值分割、Sobel 梯度、滑窗拟合以及 Flask + MJPEG 的实现方案，不影响课题总体目标的实现。
